# Contributing

Open an issue before changing cohort eligibility, schemas, exclusions, or rubric semantics. Small code and documentation fixes may go directly to a pull request.

All contributions must:

- preserve the distinction between audit errors and software failures;
- include tests for changed behaviour;
- avoid committing large or third-party artifacts;
- use British English in documentation;
- pass `make check`.

After the pilot is frozen, protocol changes require the rubric-change issue template, a dedicated pull request, and a changelog entry.
