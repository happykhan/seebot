# Limitations

- Download counts measure installations and dependency resolution, not users or scientific
  importance.
- Repository-practice observations establish presence, not effectiveness. A recognized
  test pattern does not prove test quality, and a verification workflow does not prove it
  currently passes.
- Seebot never executes upstream test suites.
- The miniature valid run checks interface behaviour and structural output only; it does
  not validate scientific correctness.
- Static analyzers report indicators, not confirmed defects or vulnerabilities.
- Analyzer values are comparable only within compatible language, analyzer, configuration,
  source-selection, and denominator groups.
- Activity recency is descriptive. Mature stable tools may need few commits.
- AI-tooling milestones are contextual annotations and cannot establish AI use or causal
  impact.
- Slow, remote-service, large-database, or specialised-hardware tools may have incomplete
  usage evidence without being project failures.
- The first execution adapter uses Pixi and Bioconda artifacts, but installer behaviour is
  not a project-health metric.
