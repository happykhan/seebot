# Limitations

- Download counts measure package downloads, not unique users or active use.
- Repository files are signals of practice, not proof that a process is effective.
- Static-analysis findings are tool-dependent observations, not defects by definition.
- Language-specific metrics do not generalise automatically to other languages. Numeric
  source comparisons require the same language and compatible configuration.
- CLI probes cannot establish scientific correctness.
- Upstream default-branch state may differ from the packaged release and is analysed separately.
- Failed audit infrastructure yields `ERROR`, never an inferred package failure.
- Agent review is bounded and sampled; confidence and adjudication are recorded.
- The Engineering Practice Award uses explicit but normative point weights. It ranks the
  selected observable practices, not overall software quality or scientific validity.
- Repository-file presence and recipe-test depth can earn award points but do not prove
  that the associated practices are effective.
- Award ranks depend on the frozen cohort and rubric version. Pilot badges are
  provisional and promotional copy is disabled until the ten-package rubric is frozen.
